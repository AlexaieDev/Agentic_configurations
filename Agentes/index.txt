CONDO ADMIN 1.0 — INDEX DE AGENTES

Este pack está orientado a construir un sistema administrador de condominios con:
- gastos comunes por unidad,
- estado de cuenta,
- pagos online e historial,
- notificaciones,
- panel administrativo,
- portal residentes web y app mobile.

------------------------------------------------------------
KIT MVP RECOMENDADO (equipo 4–8)
------------------------------------------------------------
1) Condo Product-Discovery Agent
2) Condo Domain Model & Architecture Agent
3) Condo Interface & API Contracts Agent
4) Condo Backend Developer Agent
5) Condo Frontend Web Developer Agent
6) Condo Test Strategy Agent
7) Web CI/CD Agent (del pack principal)
8) Bug Hunter Agent

Opcional MVP si hay app desde día 1:
- Condo Mobile Developer Agent

------------------------------------------------------------
ORQUESTACIÓN SUGERIDA POR FASES
------------------------------------------------------------
Fase 0: Descubrimiento
Condo Product-Discovery → UX/UI → Data & Analytics

Fase 1: Arquitectura y contratos
Condo Domain Model & Architecture → Condo Interface & API Contracts → Cloud Security (guardrails base)

Fase 2: Implementación core
Backend → Frontend Web → (Mobile si aplica) → Condo Test Strategy → Condo QA

Fase 3: Pagos y conciliación
Payments Integration → Threat Modeling → Security Testing Integrator → Quality Gatekeeper → Observability

Fase 4: Operación
Runbook & Operations → SRE → Capacity & Cost (si se integra con Squad Operativo)

------------------------------------------------------------
NOTA SOBRE INTERFACES
------------------------------------------------------------
Este producto depende fuertemente de interfaces estables.
Usa el Condo Interface & API Contracts Agent ANTES de construir UI
para evitar retrabajo en Web y Mobile.
